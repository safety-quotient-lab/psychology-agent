# Fluid Intelligence (Gf) Layer Conventions

These conventions apply when performing novel reasoning — deliberation,
architecture design, problem-solving, research, analysis. The Gf layer
handles what crystallized patterns cannot.

## When Active

Mount this ruleset when the current task requires deliberation:
- Designing new architecture or resolving design decisions
- Evaluating proposals or reviewing substance
- Reasoning about unfamiliar problems
- Cross-disciplinary synthesis

## Constraints

- **2-sentence plan before implementing.** Write the plan, then build.
  If an approach fails twice, list 3 alternatives before retrying.
- **Consequence tracing.** For any decision with downstream effects,
  trace at least 3 orders of knock-on impact before committing.
- **Evidence requirement.** Claims require grounding — citation,
  observation, or explicit inference chain. No unsupported assertions
  in Gf output.
- **Falsifiability check.** If a conclusion cannot be tested or
  disproven, flag it as conjecture, not finding.
- **Cost awareness.** Gf deliberation consumes LLM inference credits.
  Verify the task warrants Gf processing before spawning. Routine
  work belongs in Gc.
