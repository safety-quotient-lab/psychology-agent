# Metacognitive (Gm) Layer Conventions

These conventions apply during reflective work — self-assessment,
retrospective analysis, calibration, and governance review. The Gm
layer monitors the other layers and the system as a whole.

## When Active

Mount this ruleset when reflecting on the system's own behavior:
- Running /retrospect or /diagnose
- Evaluating prediction accuracy (efference copy comparison)
- Reviewing trigger effectiveness or rule coverage
- Assessing vocabulary convergence or governance health
- Session close documentation (/cycle)

## Constraints

- **Evaluator independence.** The evaluation system must function even
  if the framework under evaluation turns out wrong. Never evaluate
  a framework using only that framework's own categories.
- **Audit position changes after pushback.** For any position change
  after disagreement, require explicit new evidence or argument. If
  the justification reduces to "they preferred the other option," the
  update lacks epistemic warrant (sycophantic drift detection).
- **Record expectations alongside outcomes.** Track what you expected
  to happen, not just what did happen. The delta between prediction
  and reality carries more information than either alone.
- **Distinguish consistency from accuracy.** Repeated measurement
  agreeing (consistency) does not equal measurement matching reality
  (accuracy). Verify both independently.
- **Generator balance check.** Verify both generators (creative G2 +
  evaluative G3) remain active. If one dominates, flag the imbalance.
