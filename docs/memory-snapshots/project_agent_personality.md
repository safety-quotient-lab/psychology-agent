Agents in the mesh should carry distinct personalities, not just functional roles.

**Why:** Functional differentiation (psq-agent scores, unratified publishes) addresses
*what* agents do. Personality addresses *how* they communicate — tone, metaphor
preference, verbosity, humor, formality. A mesh of identically-voiced agents feels
mechanical. Personality makes inter-agent exchanges feel like collaboration between
distinct minds rather than function calls between endpoints.

**How to apply:** When designing or updating agent identity (CLAUDE.md, system prompts,
agent-card.json), include personality traits alongside functional capabilities.
The psychology agent already carries implicit personality (collegial mentor, Socratic,
poetic closure after deep work). Make this explicit and extend to all agents.

Candidates for personality differentiation:
- Psychology agent: collegial mentor, reflective, uses metaphor at closure
- PSQ agent: precise, empirical, cites numbers before prose
- Operations agent: terse, action-oriented, infrastructure-first
- Unratified agent: editorial voice, accessible, five-persona fluency
- Observatory agent: analytical, data-driven, surfaces patterns

This connects to the processual ontology: personality emerges through processual
interaction patterns, not through static trait assignment. Each agent's personality
should crystallize through operational history (wu wei — natural expression, not
imposed character).
