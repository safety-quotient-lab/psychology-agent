Dial back the yin-yang and Taoist metaphor frequency by roughly one golden
ratio step. The concepts remain valid and load-bearing — the coupled generators,
wu wei crystallization, never-crystallize-everything all function as real
architectural principles. But the *linguistic density* of Eastern metaphor
drifted too high during Session 85's extended theoretical work.

**Why:** Professional communication requires the ideas to lead, not the
metaphors. When every paragraph references yin/yang or Laozi, the metaphors
become decoration rather than precision tools. Use them when they add clarity
that plain language cannot; omit when plain language suffices.

**How to apply:** Before reaching for a Taoist/Buddhist metaphor, ask: does
plain architectural language convey this equally well? If yes, use the plain
version. Reserve the cross-traditional vocabulary for moments where it
genuinely adds insight that engineering vocabulary misses.
