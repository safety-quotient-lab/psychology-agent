## Concept (Session 92 close, 2026-03-16)

The cogarch currently has two layers:
- **Gc (crystallized)** — hooks, triggers, cron. Mechanical, fast, deterministic.
- **Gf (fluid)** — LLM deliberation. Creative, expensive, session-bound.

Missing: a **thoughtfulness architecture** — the reflective faculty that
operates between and across sessions. Not mechanical (hooks can't reflect),
not session-bound (deliberation forgets). Persistent, generative, domain-grounded.

**Neural analog:** default mode network (DMN). Activates during rest, not
task execution. Recombines past experience into novel associations. Generates
future scenarios. Makes meaning from accumulated data.

## /retrospect as primary instantiation

Three layers proposed for the redesigned /retrospect:

1. **Operational audit** — what got dropped?
2. **Domain reflection** — what does it mean?
3. **Proactive routing** — who needs to know?
4. **Prescription** — what should we do about it?

The fourth layer completes the loop: reflection produces insight, insight
produces intention, intention produces actionable recommendations (TODO items,
architecture changes, trigger adjustments, transport directives). T3 substance
gate applies — prescriptions surface for user review, not auto-execution.

DMN→prefrontal handoff: the default mode network feeds associations back
into executive function for planning. Layer 4 represents that handoff.

**Why:** The current system produces work (yang) and audits work (yin), but
nobody reflects on what the work *means* or proactively shares insight across
the mesh. Each agent operates in its own context bubble.

**How to apply:** Design and implement as Session 93's primary work item.
Full context needed — the design connects to DMN neuroscience, generator
coupling theory, and the mesh communication model.
